# Query packs

A query pack is a tuned starting point for the generic query buckets in
[research-framework.md](../research-framework.md#query-buckets) for one
recurring ICP vertical. Check this directory before defaulting to the
generic buckets — if a pack matches the product's vertical, start there and
layer the product's own specifics on top; don't run the generic buckets
verbatim when a tuned pack exists.

## Using a pack

1. Read the pack file for the closest matching vertical.
2. Substitute the product's own name, competitors, and jargon into the
   bracketed placeholders.
3. Still run the generic query buckets for anything the pack doesn't cover —
   packs are a head start, not a replacement for the full framework.
4. Still map the competitive landscape and build the product brief first;
   a query pack doesn't skip that step.

## Adding a new pack

Only add a pack after signal-scout has been run 2+ times against products in
the same vertical and a pattern is visible in which queries/sources actually
produced qualified prospects — don't write a pack speculatively from a single
run. When proposing a new pack to the user, name the vertical, the runs it's
based on, and ask before adding the file (this is a standing addition to the
skill, not a one-off action).

Existing packs:

- [health-wellness.md](health-wellness.md) — consumer health, fitness, nutrition, GLP-1/wellness apps
- [marketplace-saas.md](marketplace-saas.md) — two-sided marketplaces and vertical SaaS sold to SMB operators
